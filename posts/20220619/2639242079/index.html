[Angular] 響應式表單 (Reactive forms) 筆記

發表於 2022-06-19 分類於 angular

前言

這篇單純是將 angular 官網關於響應式表單的內容節錄下來的筆記

FormControl

  • 先在 .ts 透過 new 一個 FormControl 實例
  • 再在 .html 透過 formControl 把實例 與 templete 中的表單控制元件綁定起來
  • 綁定後: view 變化 -> 通知 model, model 變化 -> 反映在 view 上
  • 透過 value 得到控件當前值
  • 透過 setValue() 修改該控件的值

FormGroup

  • 與 FormControl 類似, 透過 formGroup 將 FormGroup
  • 透過 setValue() 修改 FormGroup 會嚴格遵循表單組的結構，需整體性替換控制元件的值。
  • 使用 patchValue() 方法可以用物件中所定義的任何屬性為表單模型進行替換。

FormBuilder

FormBuilder 三個工廠方法

control() -> FormControl
group() -> FormGroup
array() -> FormArray

  • 使用 FormBuilder 時, 每個控件名稱對應到一個陣列, 陣列的第一項是該控件的初始值
  • 陣列的第二、三項是非必要的, 分別對應到控件的同步和非同步驗證器

Validating form input

Validators 提供一組內建的驗證器給 FormControl 使用

  • 表單控件若為 invalid, 該狀態會傳播到父 FormGroup 元件中, 讓該 FormGroup 狀態也變成 invalid
  • 使用 Validators 其他 static method 時如果要視空值為無效的話必須加上 Validators.required

FormArray

  • FormArray 類似 FormGroup, FormArray 管理一組 FormControl, 不過不用像 FormGroup 為每一個 FormControl 指定一個 key, 在事先不知道會有多少子 FormControl 時就適合使用 FormArray
  • get 回傳的控件型別為 AbstractControl, 所以需要顯式指定型別為 FormArray

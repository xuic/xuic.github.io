[JavaScript] RegExp Flag

發表於 2023-09-25 分類於 JavaScript

每個 flag 對應 RegExp 物件的一個屬性 (若使用了某個 flag, 則該對應屬性為 true, 反之為 false)

Flag 描述 對應屬性
d Generate indices for substring matches. hasIndices
g 全局匹配模式 global
y Perform a “sticky” search that matches starting at the current position in the target string. sticky
i 不分大小寫 ignoreCase
m 讓 ^ / $ 能匹配 newline characters. multiline
s 讓 . 能匹配 newline characters. dotAll
u 啟用 unicode 模式, 字元以 code point 為單位 unicode
v u mode 的升級版 unicodeSets

d

  • 影響 exec() 的返回值 (如果用了 d flag 會多一個 indices 屬性)

    const re = /^(\d+(?:\.\d+)?) *(KB|MB|GB)$/d;
re.exec("1.2 MB");

// ['1.2 MB', '1.2', 'MB', index: 0, input: '1.2 MB', groups: undefined, indices: Array(3)]
// 其中 indices: [[0, 6], [0, 3], [4, 6], groups: undefined]

    indices[0], indices[1], indices[2] 分別對應了整個匹配項, 第一個 capturing group, 第二個 capturing group

g & y

  • RegExp.prototype.exec(string) 與 RegExp.prototype.test(string)

    • RegExp 物件若是有 g 或 y flag, 則該物件是有狀態的 (stateful)
    • 調用 exec() 與 test() 時會從 .lastIndex 的位置開始匹配, 並在匹配結束後更新 .lastIndex
    • 不含 g 或 y 
      const str = "##-#";
const re = /#/;

re.exec(str); // ['#', index: 0, input: '##-#', groups: undefined]
re.exec(str); // ['#', index: 0, input: '##-#', groups: undefined]
    • g 
      const str = "##-#";
const re = /#/g;

re.exec(str); // ['#', index: 0, input: '##-#', groups: undefined]
re.exec(str); // ['#', index: 1, input: '##-#', groups: undefined]
re.exec(str); // ['#', index: 3, input: '##-#', groups: undefined]
re.exec(str); // null
    • y 
      const str = "##-#";
const re = /#/y;

re.exec(str); // ['#', index: 0, input: '##-#', groups: undefined]
re.exec(str); // ['#', index: 1, input: '##-#', groups: undefined]
re.exec(str); // null
    • gy 與 y 的結果相同

  • String.prototype.search(regExp) 與 String.prototype.split(regExp)

    • 忽略 g 與 y

  • String.prototype.replace(regExp) 與 String.prototype.match(regExp)

    • 不含 g 或 y 
      const str = "##-#";
const re = /#/;

str.replace(re, "x"); // 'x#-#'
str.replace(re, "x"); // 'x#-#'
    • g (不會更新 .lastIndex) 
      const str = "##-#";
const re = /#/g;

str.replace(re, "x"); // 'xx-x'
re.lastIndex; // 0
    • y (更新 .lastIndex) 
      const str = "##-#";
const re = /#/y;

str.replace(re, "x"); // 'x#-#'
re.lastIndex; // 1
    • gy (不會更新 .lastIndex) 
      const str = "##-#";
const re = /#/gy;

str.replace(re, "x"); // 'xx-#'
re.lastIndex; // 0

  • String.prototype.replaceAll(regExp) 與 String.prototype.matchAll(regExp)

    • regExp 沒有 g flag 的話會拋出錯誤
    • g (不會更新 .lastIndex) 
      const str = "##-#";
const re = /#/g;

[...str.matchAll(re, "x")];
/*
[
    ['#', index: 0, input: '##-#', groups: undefined], 
    ['#', index: 1, input: '##-#', groups: undefined], 
    ['#', index: 3, input: '##-#', groups: undefined]
]
*/
re.lastIndex; // 0
    • gy (不會更新 .lastIndex) 
      const str = "##-#";
const re = /#/gy;

[...str.matchAll(re, "x")];
/*
[
    ['#', index: 0, input: '##-#', groups: undefined], 
    ['#', index: 1, input: '##-#', groups: undefined]
]
*/
re.lastIndex; // 0

i

  • 僅考慮 ASCII 字元時可簡單理解為不分大小寫, 實際原理可參考 Canonicalize 流程
  • 簡單來說 unicode 模式下使用 toCasefold, 反之則使用 toUpperCase 
    /[a-z]/i.test("ſ"); // false
/[a-z]/iu.test("ſ"); // true

m

  • 使用 m flag 讓 ^ / $ 能匹配 Line Terminators 
    const re1 = /^abc$/m;
const re2 = /^abc$/;
const str = "123\nabc\rxxx";
re1.exec(str); // ['abc', index: 4, input: '123\nabc\rxxx', groups: undefined]
re2.exec(str); // null

s

  • s 的由來是 singleline (但事實上可以跟 multiline 共用, dotAll 更準確地描述了該 flag 的功能)
  • 使用 s flag 讓 . 能匹配 Line Terminators 
    const re1 = /abc./s;
const re2 = /abc./;
const str = "123\nabc\rxxx";
re1.test(str); // ture
re2.test(str); // false

u

  • 使用 u flag 讓
    1. Unicode code point escapes (\u{xxxx}, \p{UnicodePropertyValue}) 被正確當成 Unicode code point escapes 而非 identity escapes 
      // /\u{61}/ 被視作 /u{61}/
/\u{61}/.test("u".repeat(61)); // true
/\u{61}/.test("a"); // false

// 使用 u flag 後能正確被辨識為 Unicode 字元類
/\u{61}/u.test("a"); // true
    2. 字元是以代碼點 (Code point) 為單位, 而不是以代碼單元 (Code unit) 為單位, 也就是說在 unicode 模式下代理對 (Surrogate pairs) 能被視為一個完整字元 
      /\ud83d/.test("\ud83d\ude04"); // true
/\ud83d/u.test("\ud83d\ude04"); // false

// "\ud83d\ude04" === "😄"
// 在 unicode 模式下, "\ud83d\ude04" 被視為一個完整的字元
      const re1 = /./g;
re1.test("\ud83d\ude04"); // true
re1.lastIndex; // 1
re1.test("\ud83d\ude04"); // true

const re2 = /./gu;
re2.test("\ud83d\ude04"); // true
re2.lastIndex; // 2
re2.test("\ud83d\ude04"); // false

// 在 unicode 模式下, . 匹配 code point 而非 code unit

v

  • v 代表 u 的升級版 (注意 u 與 v 兩個 flag 不能同時使用)
  • 使用 v flag 除了啟用 unicode 模式之外, 還有

    1. 讓字元類 \p{…} 能匹配字元串的屬性, 而非單個字元(代碼點)

      const re = /^\p{Emoji}$/u;
re.test("⚽"); // true, "⚽" === "\u26BD"
re.test("👨🏾‍⚕️"); // false, "👨🏾‍⚕️" === "\u{1F468}\u{1F3FE}\u200D\u2695\uFE0F"

      因為 Emoji 是 Unicode ‘字元’屬性, 而 "👨🏾‍⚕️" 是由多個字元組成, 並沒有 Emoji 屬性, 所以上面的正規表達式與 "👨🏾‍⚕️" 並不匹配。

      Unicode 標準中除了字元屬性之外, 還有一類字元串(包含一個或多個字元)屬性, 以 RGI_Emoji 屬性為範例:

      const re = /^\p{RGI_Emoji}$/v;
re.test("⚽"); // true, "⚽" === "\u26BD"
re.test("👨🏾‍⚕️"); // false, "👨🏾‍⚕️" === "\u{1F468}\u{1F3FE}\u200D\u2695\uFE0F"

    2. 能夠嵌套字符類、對字符類/字串字面值(string literal)使用差集/交集/並集運算

      • 差集 --
        使用 \q{…} 語法處理 👨🏾‍⚕️ 這種由多個代碼點組成的符號

        // 匹配除了 π 以外的所有希臘符號
/[\p{Script_Extensions=Greek}--π]/v.test("π"); // false

// 匹配除了 α, β, γ 以外的所有希臘符號
/[\p{Script_Extensions=Greek}--[αβγ]]/v.test("α"); // false

// 匹配除了 👨🏾‍⚕️ 以外的所有 RGI_Emoji
/^[\p{RGI_Emoji}--\q{👨🏾‍⚕️}]$/v.test('👨🏾‍⚕️'); // false

      • 交集 &&

        // 匹配所有希臘字母
/[\p{Script_Extensions=Greek}&&\p{Letter}]/v.test("π"); // true
/[\p{Script_Extensions=Greek}&&\p{Letter}]/v.test("\u{1018A}"); // false
/\p{Script_Extensions=Greek}/v.test("\u{1018A}"); // true, "\u{1018A}" : GREEK ZERO SIGN

// 匹配所有 ASCII 數字
/[\p{Number}&&\p{ASCII}]/v.test('1'); // true
/[\p{Number}&&\p{ASCII}]/v.test('①'); // false
/\p{Number}/v/v.test('①'); // true

      • 並集
        原本已經可以使用 /[\p{Letter}\p{Number}]/u 來匹配 \p{Letter} 或 \p{Number}, 透過使用 v flag, 可以透過組合 字元串/字元串屬性 使該功能變得更加強大

        const re = /^[\p{RGI_Emoji}\p{ASCII}\q{🇧🇪|abc}xyz0-9]$/v;
re.test('👨🏾‍⚕️'); // true
re.test('🇧🇪'); // true
re.test('abc'); // true
re.test('x'); // true
re.test('7'); // true

        /^[\p{RGI_Emoji}\p{ASCII}\q{🇧🇪|abc}xyz0-9]$/v 將會匹配

        • 字元串屬型 \p{RGI_Emoji}
        • 字元屬性 \p{Emoji}
        • 多碼點字元串 🇧🇪, abc
        • 單字元 x, y, z
        • 字元範圍語法 0 到 9

    3. 修正同時使用 i flag 與 ^ 時的行為

      在非 v mode 下, [^...] 實作方法為反轉 [...] 的結果 (只要 [...] 不匹配, [^...] 就匹配, 反之亦然)

      在這個情況下, /\p{Lowercase_Letter}/iu 與 /[^\P{Lowercase_Letter}]/iu 看似等效, 但實際上 /\p{Lowercase_Letter}/iu 匹配所有大寫與小寫字母, 而 /[^\P{Lowercase_Letter}]/iu 無法匹配到任何字元

      使用 v flag 時, ^ 的行為由將結果取反改為構造補集合, 因此不論是否使用了 i flag 都有 /\p{...}/v 等效 /[^\P{...}]/v 與 /[^\p{...}]/v 等效 /\P{...}/v

      /\p{Lowercase_Letter}/iu.test("x"); //true
/[^\P{Lowercase_Letter}]/iu.test("x"); //false

/\p{Lowercase_Letter}/iv.test("x"); //true
/[^\P{Lowercase_Letter}]/iv.test("x"); //true
